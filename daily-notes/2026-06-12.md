# SQA Learning Day 1 – Software Testing Life Cycle (STLC)

## What is STLC?

Software Testing Life Cycle (STLC) is a structured process followed by QA engineers to ensure that software meets quality requirements and works as expected before release.

## Phases of STLC

### 1. Requirement Analysis

The QA team reviews project requirements to identify testable features and understand testing scope.

### 2. Test Planning

A test strategy is prepared, including testing objectives, resources, schedule, tools, and risk analysis.

### 3. Test Case Development

Test cases and test data are created based on project requirements.

### 4. Test Environment Setup

The testing environment is configured to match the production or staging environment.

### 5. Test Execution

Test cases are executed, and actual results are compared with expected results.

### 6. Defect Reporting & Tracking

Any identified defects are reported, tracked, and retested after fixes.

### 7. Test Closure

The testing process is evaluated, reports are generated, and lessons learned are documented.

## Key Takeaway

STLC helps QA teams perform testing systematically, improve software quality, reduce defects, and ensure a reliable product before release.
