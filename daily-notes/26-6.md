# Software Testing Levels: Unit, Integration, System & User Acceptance Testing (UAT)

## Introduction

Software testing is performed at different levels to ensure that each component of an application works correctly before it is released to users.

The four primary testing levels are:

1. Unit Testing
2. Integration Testing
3. System Testing
4. User Acceptance Testing (UAT)

Each level has a specific objective and helps identify defects at different stages of software development.

---

## 1. Unit Testing

Unit Testing is the first level of testing where individual components or functions are tested independently.

### Objective

* Verify that a single unit of code works correctly.
* Detect defects at an early stage.

### Performed By

* Developers

### Example

Testing a function that calculates the total price of items in a shopping cart.

---

## 2. Integration Testing

Integration Testing verifies that multiple modules or components work together correctly.

### Objective

* Detect interface and communication issues between modules.
* Ensure data flows correctly.

### Performed By

* Developers or QA Engineers

### Example

Testing whether the Login module successfully communicates with the User Database.

---

## 3. System Testing

System Testing evaluates the complete application as a whole.

### Objective

* Verify that the entire system meets functional and non-functional requirements.
* Ensure all modules work together correctly.

### Performed By

* QA Engineers

### Example

Testing an e-commerce website by checking user registration, login, product search, checkout, and payment process together.

---

## 4. User Acceptance Testing (UAT)

User Acceptance Testing is the final testing phase where real users or clients verify that the software meets business requirements.

### Objective

* Ensure the application satisfies user expectations.
* Confirm that the software is ready for production.

### Performed By

* Clients
* End Users
* Business Representatives

### Example

A company asks its employees to test a newly developed HR Management System before it is officially launched.

---

## Comparison Table

| Testing Level                 | Purpose                          | Performed By        |
| ----------------------------- | -------------------------------- | ------------------- |
| Unit Testing                  | Test individual components       | Developers          |
| Integration Testing           | Test interaction between modules | Developers / QA     |
| System Testing                | Test the complete application    | QA Engineers        |
| User Acceptance Testing (UAT) | Validate business requirements   | Clients / End Users |

---

## Why These Testing Levels Matter

* Detect defects early in development.
* Improve software quality.
* Reduce the cost of fixing bugs.
* Ensure the application meets technical and business requirements.

---

## Interview Question

### Q: What is the difference between System Testing and User Acceptance Testing (UAT)?

**Answer:**

System Testing verifies that the complete application works according to the technical requirements, while User Acceptance Testing (UAT) verifies that the application meets the business requirements and user expectations before release.

---

## Key Takeaway

Each testing level has a unique purpose. Together, they ensure that software is technically correct, functionally complete, and ready for end users.

## Conclusion

Understanding the four levels of software testing is essential for every QA Engineer. These testing levels help deliver reliable, high-quality software while minimizing defects in production.
