# Smoke Testing vs Sanity Testing

## Introduction

Smoke Testing and Sanity Testing are important software testing techniques used to verify the stability and functionality of an application. Although they are often confused, they serve different purposes.

## What is Smoke Testing?

Smoke Testing is a preliminary test performed on a new build to ensure that the major functionalities of the application are working correctly.

### Objectives

* Verify build stability
* Ensure critical features are functional
* Decide whether further testing can continue

### Example

After deploying a new build:

* Login works
* Dashboard loads
* Navigation menus open correctly

If these critical features fail, the build is rejected.

## What is Sanity Testing?

Sanity Testing is performed after bug fixes or minor changes to verify that the specific functionality works correctly and that no related features are broken.

### Objectives

* Validate bug fixes
* Verify impacted areas
* Determine if detailed testing should proceed

### Example

A login bug was fixed:

* Test login functionality
* Check password validation
* Verify related authentication features

## Differences Between Smoke Testing and Sanity Testing

| Smoke Testing                | Sanity Testing                             |
| ---------------------------- | ------------------------------------------ |
| Performed on a new build     | Performed after bug fixes or minor changes |
| Covers major functionalities | Focuses on specific modules                |
| Broad and shallow testing    | Narrow and deep testing                    |
| Ensures build stability      | Ensures functionality correctness          |
| Done before detailed testing | Done after changes are made                |

## Advantages of Smoke Testing

* Saves testing time
* Detects major issues early
* Prevents testing unstable builds

## Advantages of Sanity Testing

* Verifies bug fixes quickly
* Ensures recent changes work correctly
* Reduces unnecessary regression testing

## Real-World Scenario

### Smoke Testing

A new version of an e-commerce website is released. QA checks:

* User registration
* Login
* Product search
* Add to cart
* Checkout

### Sanity Testing

A bug in the checkout process is fixed. QA specifically tests:

* Checkout functionality
* Payment process
* Order confirmation

## Key Takeaway

Smoke Testing checks whether the application is stable enough for testing, while Sanity Testing checks whether a specific fix or change works correctly.

## Conclusion

Both Smoke Testing and Sanity Testing help improve software quality and save testing effort. Understanding when and where to use them is essential for every Software Quality Assurance (SQA) Engineer.
