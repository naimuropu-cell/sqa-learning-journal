# Test Case vs Test Scenario

## Introduction

Test Cases and Test Scenarios are fundamental concepts in Software Quality Assurance (SQA). Both are used during testing, but they serve different purposes.

## What is a Test Scenario?

A Test Scenario is a high-level description of what needs to be tested. It represents a possible functionality or user action within the application.

### Example

Feature: Login System

Test Scenarios:

* Verify user login with valid credentials.
* Verify user login with invalid credentials.
* Verify password reset functionality.
* Verify logout functionality.

A test scenario answers:

**"What should be tested?"**

---

## What is a Test Case?

A Test Case is a detailed document that contains steps, test data, expected results, and actual results used to verify a specific functionality.

### Example

**Test Case ID:** TC_LOGIN_001

**Title:** Verify login with valid credentials

**Precondition:**
User account exists.

**Test Steps:**

1. Open login page.
2. Enter valid email.
3. Enter valid password.
4. Click Login.

**Expected Result:**
User is redirected to the dashboard.

A test case answers:

**"How should it be tested?"**

---

## Differences Between Test Case and Test Scenario

| Test Scenario              | Test Case                  |
| -------------------------- | -------------------------- |
| High-level testing idea    | Detailed testing document  |
| Defines what to test       | Defines how to test        |
| Covers broad functionality | Covers specific validation |
| Less detailed              | Highly detailed            |
| Created before test cases  | Created after scenarios    |

---

## Real Example

### Test Scenario

Verify user registration functionality.

### Possible Test Cases

1. Register with valid information.
2. Register with an existing email.
3. Register without entering required fields.
4. Register with an invalid email format.
5. Register using a weak password.

One scenario can have multiple test cases.

---

## Benefits of Test Scenarios

* Easy to create
* Improves test coverage
* Helps identify testing scope

## Benefits of Test Cases

* Provides detailed validation steps
* Improves consistency in testing
* Useful for regression testing

---

## Interview Question

### Q: Can one Test Scenario have multiple Test Cases?

**Answer:**
Yes. A single test scenario can contain multiple test cases that validate different conditions and outcomes of the same functionality.

---

## Key Takeaway

A Test Scenario defines what needs to be tested, while a Test Case defines how it should be tested. Both are essential for effective software testing and quality assurance.

## Conclusion

Understanding the difference between Test Scenarios and Test Cases helps QA engineers design better test coverage and execute testing more efficiently.
