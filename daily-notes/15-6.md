# Bug Life Cycle in Software Testing

## Introduction

A Bug Life Cycle (Defect Life Cycle) is the process that a software defect follows from identification to closure. It helps QA teams and developers track, manage, and resolve issues systematically.

## Bug Life Cycle Stages

### 1. New

The tester discovers a defect and submits a bug report.

### 2. Assigned

The defect is assigned to a developer for analysis and resolution.

### 3. Open

The developer reviews the bug and starts working on it.

### 4. Fixed

The issue has been resolved by the developer and is ready for testing.

### 5. Retest

The QA engineer verifies whether the fix works correctly.

### 6. Verified

The defect is successfully fixed and confirmed by the tester.

### 7. Closed

The bug is officially closed after successful verification.

## Additional Bug States

### Reopened

The issue still exists after the fix and is reopened by QA.

### Rejected

The reported issue is not considered a valid defect.

### Deferred

The defect is valid but will be fixed in a future release.

### Duplicate

The same issue has already been reported.

## Bug Flow

New → Assigned → Open → Fixed → Retest → Verified → Closed

## Important Bug Attributes

Every bug report should include:

* Bug ID
* Title
* Description
* Environment
* Steps to Reproduce
* Expected Result
* Actual Result
* Severity
* Priority
* Reporter Name
* Screenshot or Evidence

## Severity Levels

### Critical

System crash, data loss, security failure.

### Major

Core functionality is not working.

### Minor

A feature works but has small issues.

### Trivial

Cosmetic or UI-related problems.

## Priority Levels

### High

Must be fixed immediately.

### Medium

Important but not urgent.

### Low

Can be fixed in a future release.

## Best Practices for Bug Reporting

1. Use a clear and concise title.
2. Provide exact reproduction steps.
3. Include screenshots when possible.
4. Mention the test environment.
5. Avoid vague descriptions.
6. Verify the bug before reporting.

## Example Bug Report

**Title:** Login button does not respond after entering valid credentials.

**Steps to Reproduce:**

1. Open the login page.
2. Enter valid email and password.
3. Click the Login button.
4. check again

**Expected Result:**
User should be redirected to the dashboard.

**Actual Result:**
Nothing happens after clicking the button.

**Severity:** Major

**Priority:** High

## Conclusion

An effective Bug Life Cycle ensures that defects are properly tracked, resolved, and verified. Strong bug reporting skills are one of the most important abilities for a Software Quality Assurance (SQA) Engineer.
