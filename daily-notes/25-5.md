# Entry Criteria vs Exit Criteria in Software Testing

## Introduction

In Software Testing, Entry Criteria and Exit Criteria help determine when a testing phase should start and when it can be considered complete.

These criteria ensure that testing activities are performed in a structured and controlled manner.

---

## What is Entry Criteria?

Entry Criteria refers to the set of conditions that must be satisfied before a testing phase can begin.

It ensures that the testing team has everything required to perform testing effectively.

### Common Entry Criteria

* Requirements are finalized and approved
* Test Plan is prepared
* Test Cases are written and reviewed
* Test Environment is available
* Test Data is prepared
* Build is deployed successfully

### Example

Before System Testing begins:

✔ Requirements approved

✔ Test cases completed

✔ Test environment ready

✔ Stable build available

Only after these conditions are met can testing start.

---

## What is Exit Criteria?

Exit Criteria refers to the set of conditions that must be satisfied before a testing phase can be completed.

It helps determine whether testing objectives have been achieved.

### Common Exit Criteria

* All planned test cases executed
* Critical defects fixed
* No blocker defects remain
* Test coverage target achieved
* Test summary report completed
* Stakeholder approval received

### Example

Before releasing software:

✔ 100% critical test cases executed

✔ No open critical defects

✔ Regression testing passed

✔ Test report approved

Only then can testing be closed.

---

## Entry Criteria vs Exit Criteria

| Entry Criteria                  | Exit Criteria                 |
| ------------------------------- | ----------------------------- |
| Conditions to start testing     | Conditions to finish testing  |
| Checked before testing begins   | Checked before testing ends   |
| Focuses on readiness            | Focuses on completion         |
| Ensures resources are available | Ensures quality goals are met |

---

## Why Are They Important?

### Benefits of Entry Criteria

* Prevents testing unstable builds
* Improves testing efficiency
* Ensures proper preparation

### Benefits of Exit Criteria

* Maintains software quality
* Ensures testing objectives are met
* Reduces production risks

---

## Real-World Scenario

### Entry Criteria

A QA team receives a new build for testing.

Before starting:

* Requirements reviewed
* Test environment configured
* Test cases approved

Testing begins.

### Exit Criteria

After testing:

* All test cases executed
* No critical defects open
* Test report submitted

Testing is completed.

---

## Interview Question

### Q: What is the difference between Entry Criteria and Exit Criteria?

**Answer:**

Entry Criteria defines the conditions required to start testing, while Exit Criteria defines the conditions required to complete testing successfully.

---

## Key Takeaway

Entry Criteria ensures testing starts at the right time with the right resources, while Exit Criteria ensures testing ends only after quality objectives have been achieved.

## Conclusion

Both Entry Criteria and Exit Criteria are essential for effective test management. They help QA teams control testing activities, improve software quality, and reduce project risks.
