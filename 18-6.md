# HTTP Methods in API Testing

## Introduction

HTTP Methods define the type of action a client wants to perform on a server. Understanding HTTP methods is essential for API testing because they determine how data is created, retrieved, updated, or deleted.

## Common HTTP Methods

### 1. GET

The GET method is used to retrieve data from a server.

#### Example

Retrieve a list of users:

```http
GET /users
```

#### Expected Result

* Status Code: 200 OK
* Returns requested data

#### Characteristics

* Read-only operation
* Does not modify data

---

### 2. POST

The POST method is used to create new data on the server.

#### Example

```http
POST /users
```

#### Request Body

```json
{
  "name": "John",
  "email": "john@example.com"
}
```

#### Expected Result

* Status Code: 201 Created
* New resource is created

---

### 3. PUT

The PUT method is used to update an existing resource completely.

#### Example

```http
PUT /users/1
```

#### Expected Result

* Status Code: 200 OK
* Existing resource updated

---

### 4. PATCH

The PATCH method is used to partially update an existing resource.

#### Example

```http
PATCH /users/1
```

#### Request Body

```json
{
  "email": "newemail@example.com"
}
```

#### Expected Result

* Only specified fields are updated

---

### 5. DELETE

The DELETE method is used to remove a resource from the server.

#### Example

```http
DELETE /users/1
```

#### Expected Result

* Status Code: 200 OK or 204 No Content
* Resource removed successfully

---

## HTTP Methods Summary

| Method | Purpose                |
| ------ | ---------------------- |
| GET    | Retrieve data          |
| POST   | Create new data        |
| PUT    | Update entire resource |
| PATCH  | Update specific fields |
| DELETE | Remove data            |

---

## API Testing Checklist

When testing an API endpoint, verify:

* Correct status code is returned
* Response body contains expected data
* Response time is acceptable
* Invalid inputs are handled properly
* Authentication works correctly
* Error messages are meaningful

---

## Interview Question

### Q: What is the difference between PUT and PATCH?

**Answer:**

PUT updates the entire resource, while PATCH updates only specific fields of a resource.

Example:

PUT:

* Updates all user information.

PATCH:

* Updates only the email address.

---

## Key Takeaway

HTTP methods are the foundation of API communication. A QA Engineer must understand when and how each method is used to effectively test APIs and identify defects.

## Conclusion

Mastering HTTP methods helps QA engineers perform accurate API testing, validate backend functionality, and ensure reliable communication between applications and servers.
