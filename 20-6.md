# Verification vs Validation in Software Testing

## Introduction

Verification and Validation are two important quality assurance activities in software development. Although they are closely related, they serve different purposes.

A simple way to remember:

**Verification:** Are we building the product right?

**Validation:** Are we building the right product?

---

## What is Verification?

Verification is the process of evaluating work products such as requirements, design documents, and code without executing the software.

Its purpose is to ensure that the product is being developed according to specifications and standards.

### Examples of Verification Activities

* Requirement Review
* Design Review
* Code Review
* Walkthroughs
* Inspections

### Characteristics

* Static process
* Does not require code execution
* Performed early in the development lifecycle
* Helps prevent defects

---

## What is Validation?

Validation is the process of evaluating the actual software by executing it to ensure it meets user needs and business requirements.

Its purpose is to confirm that the developed software works as expected.

### Examples of Validation Activities

* Functional Testing
* Integration Testing
* System Testing
* User Acceptance Testing (UAT)

### Characteristics

* Dynamic process
* Requires software execution
* Performed after development activities
* Helps detect defects

---

## Verification vs Validation

| Verification                                  | Validation                                |
| --------------------------------------------- | ----------------------------------------- |
| Checks whether the product is built correctly | Checks whether the right product is built |
| Static process                                | Dynamic process                           |
| No code execution required                    | Requires code execution                   |
| Focuses on process and documents              | Focuses on the final product              |
| Prevents defects                              | Detects defects                           |

---

## Real-World Example

### Verification

A QA engineer reviews the requirement document and confirms that the login feature must support email and password authentication.

### Validation

The QA engineer tests the login page to verify that users can successfully log in using valid credentials.

---

## Benefits of Verification

* Reduces development errors
* Improves requirement understanding
* Saves testing effort
* Detects issues early

## Benefits of Validation

* Ensures software meets user expectations
* Identifies functional defects
* Improves product quality
* Increases customer satisfaction

---

## Interview Question

### Q: What is the main difference between Verification and Validation?

**Answer:**

Verification checks whether the software is being built according to requirements and standards, while Validation checks whether the final software satisfies user needs and works correctly.

---

## Key Takeaway

Verification focuses on reviewing and evaluating development artifacts without executing the software, whereas Validation focuses on testing the actual software through execution.

Both are essential for delivering high-quality software products.

## Conclusion

Verification and Validation work together to ensure software quality. Verification helps prevent defects during development, while Validation helps detect defects before release.
